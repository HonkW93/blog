使用说明

使用说明

部分功能由于与插件相关联，请参考终篇效果。这里使用说明只介绍automatic.vim修改的部分功能。

AutoInst

操作步骤

  1. 写标志/*autoinst*/

    将要例化的模块写为如下格式：module_name inst_name(/*autoinst*/);

    • 注意格式末尾必须要有分号;
    • 注意格式可不在同一行，只要(/*autoinst*/)包含在括号内，括号外为module_name inst_name即可；支持端口处带parameter的写法

    e.g.

  2. 自动例化

    • 使用菜单栏点击AutoInst(0)或在命令行输入:call AutoInst(0)确认，进行/*autoinst*/当前模块自动例化，注意例化时光标必须置于/*autoinst*/所在行之前的位置（即在当前行或上一行，若在当前行则必须在/*autoinst*/所在列之前）；
    • 使用菜单栏点击AutoInst(1)或在命令行输入:call AutoInst(1)确认，进行/*autoinst*/所有模块自动例化；

特性

  1. 跨文件夹例化

    在代码中添加如下格式的内容声明文件夹即可：

    • verilog-library-directories为要选择的文件夹，文件夹之间以空格隔开；
    • verilog-library-directories-recursive为是否进行文件夹递归搜索，即搜索所有选择文件夹及其子文件夹；

    • 假设需要选择当前文件夹以及其下所有子文件夹作为例化搜索的对象，则配置为：

      //verilog-library-directories:(".")
      //verilog-library-directories-recursive:1

    • 注意如果使用递归搜索，请勿采用重叠的文件夹，否则递归会报错；例如上述例子中，"." 当前文件夹递归搜索包含"./aaa/bbb/ccc"子文件夹，若此时使用递归搜索则重复调用时会报错。

  2. 添加//INST_NEW //INST_DEL以及io_dir

    • 例化时默认自动添加io_dir，即端口类型input/output/inout

    • 例化时默认若有端口更新，自动在该端口尾部添加//INST_NEW

    • 例化时默认若有端口被删除，自动在所有端口例化之后添加//INST_DEL

    可在脚本automatic.vim中如下位置配置相关参数选择不添加以上3种注释

  3. 重刷

    自动保留/*autoinst*/上方的例化端口，只对其余端口进行自动例化。

AutoPara

操作逻辑基本同AutoInst一致，标志为/*autoinstparam*/。

操作步骤

  1. 写标志/*autoinstparam*/
  2. 自动例化参数

特性

  1. 支持跨文件夹例化参数

  2. 支持添加//PARA_NEW //PARA_DEL，并支持配置使用端口参数例化或使用所有参数进行例化

    可在脚本automatic.vim中如下位置配置相关参数选择不添加//PARA_NEW //PARA_DEL，并可通过配置ONLY_PORT确定使用哪种参数进行例化

  3. 支持重刷

  4. 支持特殊的parameter连续多个的写法，例如parameter A = 1, B = 5, C = 6

对齐

可在脚本automatic.vim中如下位置配置相关参数选择自动例化时对齐的位置：

name_pos_max：信号名对齐位置

symbol_pos_max：第二个括号对齐位置

start_pos：起始位置

如果信号长度过长（长于设定的对齐位置），则会按照最长信号长度+4为新的对齐位置进行对齐。

原函数梳理

下方为automatic的函数列表关系。

下方梳理函数关系，以便优化代码供个人使用

主函数

自动例化（AutoInst）

KillAutoInst

功能

删除所有自动例化内容，只保留module xxx u_xxxx (/*autoinst*/);样式的内容

参数

kill_all

  • 0：关闭第一个例化
  • 1：关闭所有例化

  • 搜索/*autoinst，搜索到后将当前行的*/及其之后的内容 替换为*/);

  • 如果是多行例化，则搜索);结尾的行，搜索到即结束。若遇到结尾$或endmodule，也直接结束搜索。跳过多行例化的所有内容

  • UpdateBuf更新数据

AutoInst

功能

自动例化，分为单行（oneline）和多行模式。

参数

kill_all

  • 0：只例化当前行
  • 1：例化所有可例化的行
  • 获取例化模块名和其文件名关系
    • 若kill_all为0，只例化当前行，则通过GetInst获取当前行的{inst : inst_file}（例如uart : uart.v）格式的insts字典
    • 若kill_all为1，例化所有可例化的行，则通过GetInsts获取所有行的{inst : inst_file}（例如uart : uart.v）格式的insts字典
  • 通过KillAutoInst删除当前所有已例化的内容，为重新例化进行准备
  • 开始循环，通过kill_all控制循环次数
  • 搜索/*autoinst行，通过GetInstNameAdvance获取例化(inst_name)模块名及例化名（u_inst）（例如uart : u_uart）；通过与insts字典的比对搜索inst_name在insts中是否有文件inst_file存在
  • 若有，则读此文件inst_file，否则报错
  • 通过GetSeqIO获取inst_file的端口列表inst_io_seq
  • 通过端口列表inst_io_seq以及GetPortMaxLen获取端口最大长度max_len，包括prefix_max_len以及suffix_max_len，用于生成prefix_margin以及suffix_margin
  • 单独判断oneline_mode，生成单行的例化格式
  • 否则为多行的例化，按格式输出例化后的端口。格式为：' .'.sig.prefix_margin.'('.sig.'['.width.'].suffix_margin.')'.','.io_dir(存在width则添加，否则不加；同理，最后一个端口添加,否则不添加)
  • UpdateBuf更新数据
  • 注意：AutoInst只接受//类型的端口注释，不接受/**/类型的端口注释

AutoInstUpdate

基本操作类似AutoInst，功能是为新添加的端口添加注释' // INST_NEW'，为已删除的端口添加注释' // INST_DEL: port ' . l:port . ' have deleted'

AutoInstUpdateOrder

基本操作类似AutoInst，功能是更新现有端口的顺序

AutoInstPortReAlign

基本操作类似AutoInst，功能是对齐现有端口排列

AppendPortDriectionToInst

基本操作类似AutoInst，功能是在尾部添加'// ' .io_dir端口方向（input/output/inout）

辅助函数

自动例化（AutoInst）辅助

GetInsts

功能

获取所有例化的模块名及路径。

  • SkipCommentLine跳过注释行
  • 匹配包含autoinst的行；通过GetInstNameAdvance获取例化(inst)模块名及例化名（u_inst）（例如uart : u_uart）；通过GetInstFileName获取包含例化模块的文件名（例如uart.v）
  • 按{inst : inst_file}（例如uart : uart.v）的格式放入名为insts的字典中

GetInst

功能

获取当前行的单个例化的模块名及路径。

功能同Getinsts，只是仅获取当前行。

GetInstNameAdvance

功能

获取例化模块名及例化名。(例如uart : u_uart)

  • SkipCommentLine跳过注释行

  • 向上搜索/*autoinst

    • 如果搜索到(/*autoinst，则为当行例化，check_autoinst_pair=1；否则即为/*autoinst，则wait_autoinst_pair=1。

    • 如果wait_autoinst_pair==1，则向上搜索(的行，若搜索到(行则check_autoinst_pair=1，且将

      否则报()匹配错误

    • 以上两种情况分别对应以下两种写法：

    • 当check_autoinst_pair==1，则当前行为(行，result为(所在列，将光标置于(位置，通过searchpairpos()搜索其匹配的)位置，然后wait_inst_name=1

    • 当wait_inst_name==1， 则向上搜索词，搜索到的第一个词为inst_name，然后删除当前行的inst_name及其前部空格（用于处理模块名module_name和例化名inst_name在同一行的情况，如下），最后置wait_module_name=1

    • 若在模块名module_name和inst_name在同一行，则处理后可直接获取模块名module_name，否则保持wait_module_name=1，继续往上搜索至模块名module_name。期间如果遇到#(.parameter(parameter))的相关行，则直接跳过(情况如下)。

GetInstFileName

功能

根据输入的例化模块名(inst)，获取例化模块的文件名。(例如uart 的文件名uart.v)

  • 若存在tags文件，则根据tags进行例化文件关系的标记，否则搜索当前文件夹./中的.v文件，使用同名文件作为例化模块的文件名
  • 若不存在文件则返回''，代表空

GetSeqIO

功能

根据输入的文本，获取端口列表。

格式为[width,type,has_defined,seq,line,signal_name,io_dir,last_port]

端口列表的格式为[width,type,has_defined,seq,line,signal_name,io_dir,last_port]，其含义分别为

  • width: 位宽，例如某端口[2:0]时width = 2，特殊符号例如``define 时width = ‘c0’`
  • type: 端口类型，正常时type = 'io_wire' 或 type = 'io_reg'，其他特殊符号时type = 'keep'
  • has_defined: 存在端口的wire或reg定义，若存在则为1
  • seq： 端口序列号，格式为五位数，不够位数的填0，例如'00001',00101
  • line：此行的原始文本
  • signal_name: 信号名称
  • io_dir: 端口方向，io_dir = 'input'或io_dir = 'output'
  • last_port: 是否为最后一个端口的标志，用于结束端口列表，丢弃错误载入端口列表的数据

操作步骤如下：

  • SkipPairCommentLine_array跳过/*...*/的多行注释

  • 去除行尾部的注释

  • 跳过module所在行

  • 其他行的处理方式分为几种：

    • 以`ifdef,`else,以及`endif开头的行，以及以//开头的注释行，记录格式为

      value = ['c0','keep',0,seq,line,seq,'',0]

    • 多个连续空白行删除为1个，其余跳过

    • 正常的输入输出端口通过ExtendIoFromLine2获得端口列表，记录格式为

      value = [width,'io_wire/io_reg',0/1,seq,'',signal_name,'input/output/inout',0]

      注意line项被置为了空

  • 搜索到always,assign，endmodule，autodef的开头行截止

  • 搜索最后一个type!='keep'的行，即搜索到最后一个正常行，将其last_port标志置为1

  • 删除结尾的空行以及开头的空行

  • 按序号排列输出端口列表的字典

ExtendIoFromLine

功能同ExtendIoFromLine2，只是端口列表以signal_name为key，即 call extend(a:dict, {signal_name : value})

ExtendIoFromLine2

功能

根据输入的文本，获取端口列表。

格式为

value = [width,'io_wire/io_reg',0/1,seq,line,signal_name,'input/output/inout',0]

对应

value = [width,type,has_defined,seq,line,signal_name,io_dir,last_port]

  • 读取line，获取对应参数，包括
    • 通过头部判断input，output或inout
    • 通过判断wire/reg确定has_defined为0/1
    • 通过判断[:]确定位宽
    • 通过删除其他部分内容后得到信号名称signal_name
    • 通过当前序号seq以及Seq2String获得序号字符串
  • 组成对应端口列表
  • 端口列表以seq为key，即 call extend(a:dict, {seq : value})

ConvertSeqIO

与ConvertIO相反，将将{signal_name : value}格式的字典转换为{seq : value}格式

ConvertIO

与ConvertSeqIO，将{seq : value}格式的字典转换为{signal_name : value}格式

其他（Others）辅助

SkipCommentLine

功能

根据输入的行号(line_idx)，搜索最新的非注释行的行号

参数

mode

  • 0：从头开始搜索
  • 1：从尾开始搜索

  • 搜索//单行注释以及/*...*/的多行注释；忽略单行的/*...*/注释，因为其可能为自动例化/*autoinst*/等命令

  • 从输入行号开始，搜索到注释行则跳过，如果非注释行则返回行号

SkipPairCommentLine_array

功能

根据输入的行号(line_idx)及内容lines，搜索最新的非/*...*/的多行注释的行号

参数

mode

  • 0：从头开始搜索
  • 1：从尾开始搜索
  • 搜索/*...*/的多行注释；忽略单行的/*...*/注释，因为其可能为自动例化/*autoinst*/等命令
  • 从输入行号开始，搜索到/*...*/的多行注释则跳过，如果非注释行则返回行号

Seq2String

功能

当前序号seq转换为字符串

格式为五位数，例如5->'00005',103->'00103'

CalMargin

功能

计算剩余空格位置

  • 如果max_len > cur_len则添加max_len-cur_len+1个空格，否则不添加空格

UpdateBuf

功能

更新当前buffer为newlines内容，鼠标跳转至goto_lnum的位置

参数

new_lines

  • 更新的内容，按行排列于数组内

goto_lnum

  • 鼠标位置
  • 如果当前行数多余newlines行数则删除多余行，否则直接覆盖（setline()）
  • 鼠标跳转到制定的goto_lnum位置，如果goto_lnum为-1则不跳转鼠标位置
